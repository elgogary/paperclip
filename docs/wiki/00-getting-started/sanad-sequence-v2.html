Phase 1C
Evaluation Engine — Active Quality Gate
TRIGGERED · ACTIVE · ON CHANGE · SRS NEEDED NEXT
Custom Service + DeepEval
What the Eval Engine Does
When developer changes anything (prompt, model, skill, tool) — engine runs, compares before vs after, gives verdict. Like CI/CD but for agent quality.
Triggered by code change — not always-on
Pulls baseline from PostgreSQL
Runs DeepEval sidecar (Docker REST API)
Compares 5 dimensions before vs after
Outputs: promote or rollback verdict
DeepEval: Python · Docker · REST API · Apache-2.0
5 Evaluation Dimensions
What Gets Evaluated
Every dimension produces a delta score: before vs after the change.
1. Task Eval — tool correctness, plan adherence, output quality
2. Skill Eval — rolling score → skill_level in Qdrant
3. Agent Eval — trust_score, failure patterns, weak vs strong skills
4. System Eval — latency p50/p95/p99 (fed from Prometheus)
5. Model Comparison — same task, diff LLMs, cost_efficiency = score/cost
Model Routing Output
Auto Routing Table
Eval Engine produces the routing table. LLM Router uses it. Sanad auto-routes to cheapest model that meets quality threshold.
invoice_processing → GLM-4 (score 0.91, $0.02)
legal_contract → Claude (score 0.95, $0.10)
data_extraction → Gemini (score 0.89, $0.01)
→ Updated automatically after each eval run
Mentor reads skill_level delta + trust_score from here. Graduation triggered when threshold met.
